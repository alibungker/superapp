# Dokumen Desain Sistem (SDD) 
## Sistem Informasi Guru Aceh - guru.acehapp.com
### (SiGuru - Sistem Informasi Guru Unggul Provinsi Aceh)

---

## 1. Pendahuluan

### 1.1 Tujuan Dokumen
Dokumen ini menyediakan desain sistem terperinci untuk Sistem Informasi Guru Aceh (guru.acehapp.com), yang dikenal juga sebagai SiGuru (Sistem Informasi Guru Unggul), sebuah platform digital terintegrasi yang dirancang untuk mendukung peningkatan kualitas guru di Provinsi Aceh, Indonesia.

### 1.2 Ruang Lingkup Sistem
SiGuru adalah sistem manajemen komprehensif yang mencakup:
- Manajemen data pribadi dan profesional guru
- Pengelolaan portofolio pendidikan dan pelatihan
- Sistem asesmen kompetensi guru
- Pelaporan statistik tingkat daerah
- Integrasi antara berbagai stakeholder pendidikan

### 1.3 Referensi
- Peraturan Menteri Pendidikan dan Kebudayaan Republik Indonesia Nomor 2 Tahun 2017 tentang Standar Kompetensi Lulusan Pendidikan dan Tenaga Kependidikan
- Laravel Framework 11.x Documentation
- Standar Nasional Pendidikan Indonesia

### 1.4 Definisi dan Akronim
- **SiGuru**: Sistem Informasi Guru Unggul
- **PTK**: Pendidik dan Tenaga Kependidikan
- **NUPTK**: Nomor Unik Pendidik dan Tenaga Kependidikan
- **SDD**: System Design Document (Dokumen Desain Sistem)
- **SRS**: Software Requirements Specification
- **RIPH**: Rencana Induk Pengembangan Human Capital

## 2. Representasi dan Pendekatan Arsitektur

### 2.1 Arsitektur Aplikasi
Aplikasi SiGuru mengadopsi arsitektur berbasis web dengan pendekatan **Model-View-Controller (MVC)** menggunakan framework **Laravel 11.x**. Arsitektur berlapis terdiri dari:

- **Presentation Layer**: Antarmuka web berbasis Blade templates dan Tailwind CSS 
- **Application Layer**: Controllers, Middleware, dan Request Handler Laravel
- **Business Logic Layer**: Eloquent Models dan Service Classes
- **Data Access Layer**: Eloquent ORM dan Query Builder
- **Data Layer**: Database MySQL dengan skema normalisasi tinggi

### 2.2 Teknologi dan Platform
- **Backend Framework**: PHP 8.2+ dengan Laravel 11.x
- **Database**: MySQL 8.0+
- **Frontend**: HTML5, CSS3 (Tailwind CSS), JavaScript
- **Server**: Linux (Ubuntu/Debian) dengan Apache/Nginx
- **Package Manager**: Composer

### 2.3 Pola Desain dan Prinsip
- **Model-View-Controller (MVC)**: Memisahkan logika bisnis, representasi data, dan antarmuka
- **Active Record Pattern**: Eloquent ORM untuk interaksi database
- **Repository Pattern**: Abstraksi akses data
- **Role-Based Access Control (RBAC)**: Sistem otorisasi berbasis peran
- **Polymorphic Relations**: Fleksibilitas struktur data
- **Soft Delete**: Pembuatan audit trail dan rekonsiliasi data

## 3. Struktur Data Spesifik Konteks Indonesia

### 3.1 Skema Database Indonesia
Database SiGuru dirancang khusus untuk konteks pendidikan Indonesia dengan struktur referensial yang kuat:

**Tabel Master Regional:**
- `cities`: 23 kabupaten/kota di Aceh (kode wilayah KEMENDIKBUD)
- `districts`: Kecamatan dalam wilayah Aceh
- `education_units`: Jenis satuan pendidikan (SD, SMP, SMA, SMK, SLB)

**Tabel Kepemilikan Data:**
- `users`: Akun sistem berdasarkan hak akses
- `profiles`: Data pribadi (NIK, nama lengkap, tempat/tanggal lahir, jenis kelamin)
- `teachers`: Data profesional dan kepegawaian guru
- `schools`: Data institusi pendidikan

**Tabel Operasional:**
- `assesment_results`: Hasil asesmen kompetensi guru
- `education_histories`: Riwayat pendidikan guru
- `training_histories`: Riwayat pelatihan dan diklat
- `planning_histories`: Rencana pelaksanaan pembelajaran (RPP)

### 3.2 Model Utama dan Relasi Spesifik

#### 3.2.1 Model User
- **Deskripsi**: Entitas akun sistem SiGuru
- **Atribut**: id, profile_id, role_id, username, email, password
- **Relasi**: BelongsTo(Role), BelongsTo(Profile)

#### 3.2.2 Model Profile
- **Deskripsi**: Data pribadi wajib sesuai registrasi KTP
- **Atribut**: 
  - id (bigint unsigned, auto increment)
  - profilehas_type, profilehas_id (polymorphic ke Teacher)
  - nik (varchar 17, unique) - Nomor Induk Kependudukan
  - slug (varchar 255, unique) - URL-friendly identifier
  - nama_lengkap (varchar 255)
  - jenis_kelamin (enum L/P) - Laki-laki/Perempuan
  - tempat_lahir, tanggal_lahir - Data registrasi KTP
  - no_hp (varchar 20) - Kontak
- **Relasi**: MorphTo(Teacher)

#### 3.2.3 Model Teacher
- **Deskripsi**: Data kepegawaian dan profesional guru
- **Atribut**:
  - id (bigint unsigned, auto increment)
  - employee_status_id, occupation_id, level_id (foreign keys)
  - nip (varchar 18) - Nomor Induk Pegawai (jika PNS)
  - nuptk (varchar 16) - Nomor Unik Pendidik dan Tenaga Kependidikan
  - jenis_ptk (enum: kepala sekolah/guru/tenaga kependidikan)
  - jenis_ptk penting karena klasifikasi sesuai Permendikbud
  - Sistem sertifikasi (sudah/belum), guru pengerak (sudah/belum)
- **Relasi**: MorphOne(Profile), BelongsToMany(School), HasMany(AssessmentResults)

## 4. Arsitektur Sistem dan Subsistem Spesifik Aceh

### 4.1 Subsistem Otentikasi Multi-Peran
**Kelas**: `App\Models\User`, `App\Http\Controllers\Auth\*`, `App\Models\Role`

**Fungsi**:
- Login/logout berbasis peran (Admin, Cabdin, Sekolah, Guru)
- Registrasi dengan proses verifikasi bertahap
- Manajemen sesi dan keamanan akses

**Desain**:
- Sistem otentikasi Laravel bawaan
- Middleware berbasis peran: 
  - `role:admin` - Akses penuh ke semua data
  - `role:cabdin` - Akses berdasarkan wilayah kerja
  - `role:sekolah` - Akses data guru di sekolah masing-masing
  - `role:guru` - Akses ke data pribadi dan portofolio

### 4.2 Subsistem Registrasi dan Verifikasi
**Proses**: `/daftar`, `/verifikasi/data`, `/daftar/akun`

**Fungsi**:
- Registrasi bertahap (step1, step2)
- Validasi data pribadi dan kepegawaian
- Pembuatan akun berdasarkan NIK dan data resmi

**Desain**:
- Sistem form bertahap dengan validasi data
- Validasi format NIK (17 digit), NUPTK (16 digit), NIP (18 digit)
- Integrasi dengan sistem data resmi pendidikan

### 4.3 Subsistem Manajemen Guru
**Kelas**: `App\Models\Teacher`, `App\Models\Profile`, `App\Http\Controllers\Back\Teacher\*`

**Fungsi**:
- Manajemen data kepegawaian dan profesional
- Manajemen portofolio pendidikan dan pelatihan
- Penyusunan rencana pembelajaran (RPP digital)

**Desain**:
- Model kompleks dengan 30+ relasi
- Sistem input terstruktur berdasarkan jenis data
- Validasi format sesuai standar pendidikan nasional

### 4.4 Subsistem Asesmen Kompetensi
**Kelas**: `App\Models\AssesmentResult`, `App\Models\Competence`, `App\Http\Controllers\Back\*`

**Fungsi**:
- Penyimpanan hasil asesmen berdasarkan kompetensi
- Pemetaan kompetensi sesuai standar nasional
- Perhitungan skor dan level kompetensi

**Desain**:
- Tabel `assesment_results` menyimpan:
  - `skor_pedagogik`, `skor_kurikulum_merdeka`, `skor_professional`
  - `total` sebagai agregasi ketiga aspek
  - Referensi ke `competence` untuk bidang spesifik
- Sistem level kompetensi: 1 (sangat rendah) hingga 6 (sangat tinggi)
- Sistem rekapitulasi berdasarkan wilayah (kota/kabupaten)

### 4.5 Subsistem Pelaporan dan Rekapitulasi
**Kelas**: `App\Models\TeacherInfo`, `App\Http\Controllers\Back\Admin\RecapController`

**Fungsi**:
- Pembuatan laporan statistik guru
- Rekapitulasi berdasarkan wilayah geografis
- Visualisasi data asesmen dan kompetensi

**Desain**:
- Tabel `teacher_infos` menyimpan data agregat:
  - `jumlah_guru`, `jumlah_guru_slb`, `jumlah_guru_sma`, dll.
  - `laki_laki`, `perempuan` - Demografi gender
  - `guru_pns`, `guru_pppk`, `guru_gty`, `guru_honor_daerah` - Status kepegawaian
  - `sertifikasi`, `belum_sertifikasi` - Status sertifikasi
  - `asesmen_level6`, `asesmen_level5`, ..., `asesmen_level1` - Distribusi level kompetensi
- Query kompleks untuk agregasi data tingkat kota/kabupaten
- Sistem filter dan pencarian berdasarkan wilayah

## 7. Desain Database Spesifik Indonesia

### 7.1 Normalisasi dan Struktur
Database SiGuru dirancang dengan prinsip normalisasi tinggi untuk memastikan konsistensi data administrasi pendidikan Indonesia:

**Tabel Master:**
- `cities` (1-23): Semua kabupaten/kota di Provinsi Aceh
- `districts`: Kecamatan dalam wilayah Aceh
- `employee_statuses`: Status kepegawaian (PNS, PPPK, Honorer, dll.)
- `occupations`: Jabatan fungsional (Guru, Kepala Sekolah, dll.)
- `competences`: Bidang kompetensi sesuai kurikulum dan standar nasional

**Tabel Transaksional:**
- `teacher_data` (29,000+ records): Data spesifik guru
- `assesment_results` (20,000+ records): Data asesmen kompetensi
- `education_histories`, `training_histories`, `planning_histories`: Portofolio guru

### 7.2 Kinerja dan Indeks
- Indeks unik pada NIK, email, NPSN untuk mencegah duplikasi
- Indeks pada foreign keys untuk query efisien
- Indeks pada kolom yang sering digunakan untuk filter geografis
- Sistem indexing untuk query agregat statistik

### 7.3 Audit dan Integritas
- Soft deletes untuk audit trail perubahan data
- Polymorphic relations untuk fleksibilitas data
- Constraints referensial untuk integritas data
- Sistem logging aktivitas kritis
